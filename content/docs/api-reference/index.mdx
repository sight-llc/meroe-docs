## What's documented here

Everything a developer building on Meroe needs to call:

- **Developer Auth** — login and token refresh (`/v1/developers/auth/login`, `/v1/developers/auth/refresh`). Registration is not included — onboarding happens via `app.meroe.dev`, not the API.
- **Apps** — create and list app namespaces that scope customers and virtual accounts
- **API Keys** — mint, list, and revoke scoped API keys programmatically
- **Customers** — create customers, provision NUBANs, query balances and identity

<Note>
  Every route under `/v1/admin/` is intentionally excluded — those are internal
  tools for the Meroe team (KYC approval, platform stats, reconciliation rule
  management, audit log), not endpoints integrators call.
</Note>

## Base URL

```
https://api.meroe.dev/v1
```

## Authentication

All endpoints except login and refresh require a bearer token:

```bash
curl https://api.meroe.dev/v1/customers \
  -H "Authorization: Bearer mr_test_sk_..."
```

See [Authentication](/docs/guides/authentication) for key scopes, rotation, and sandbox vs live keys.

## Sandbox environment

When you use a `mr_test_sk_...` key, every API call hits Nomba's real test environment (`sandbox.nomba.com`) — real NUBAN provisioning, real (test) money movement. There is no separate Meroe sandbox mode for API consumers; the `mr_test_sk_...` key is the switch.

Do not mention Spring profiles or "fake sandbox" mode — those are for backend contributors running the service locally and are not relevant to API consumers.

## Simulating an inbound payment

Send a bank transfer from any bank app to your customer's NUBAN. Nomba's sandbox accepts test transfers; once the transfer lands, Meroe's webhook listener reconciles it automatically and credits the customer's wallet.

## Webhook test endpoint

Use `POST /v1/webhook-subscriptions/{id}/test` to send a test event to your registered webhook URL without needing a real transfer. See the [API Reference](/docs/api-reference) for this endpoint.

## Sandbox deposit history

`GET /v1/sandbox/history` returns a list of simulated deposits. See the [API Reference](/docs/api-reference) for this endpoint.
